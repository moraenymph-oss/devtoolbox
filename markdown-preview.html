마크다운 미리보기

왼쪽에 마크다운을 입력하면 오른쪽에 렌더링 결과가 실시간으로 표시됩니다.

🔒 입력한 내용은 브라우저에서만 렌더링되며 서버로 전송되지 않습니다.

광고 영역

사용법

왼쪽 입력창에 마크다운 문법(# 제목, **굵게**, - 목록, ```코드블록``` 등)을 입력하면 오른쪽 미리보기 영역에 타이핑과 동시에 렌더링 결과가 나타납니다. 별도의 저장이나 미리보기 버튼 없이 바로 확인할 수 있어 문서를 작성하면서 결과물을 실시간으로 검토할 수 있습니다. HTML 복사 버튼을 누르면 렌더링된 결과의 HTML 코드를 클립보드로 복사해 블로그 에디터 등에 붙여넣을 수 있습니다. 렌더링에는 널리 쓰이는 오픈소스 라이브러리 marked.js를 사용해 GitHub 마크다운과 유사한 결과를 보여줍니다. 두 버전의 마크다운 초안을 비교하고 싶다면 텍스트 Diff 비교기를 함께 사용해 보세요.

이런 상황에서 쓰세요

  • GitHub 저장소의 README.md를 작성하면서 실제 렌더링 결과를 바로 확인할 때
  • 이슈나 PR 코멘트에 마크다운 서식을 넣기 전에 결과를 미리 볼 때
  • 기술 블로그나 위키 문서 초안을 마크다운으로 작성할 때
  • 노션·옵시디언 등에서 옮겨 온 마크다운이 제대로 표시되는지 확인할 때
  • 표, 코드 블록, 인용문 등 복잡한 마크다운 문법을 테스트할 때

입력한 마크다운은 브라우저 안에서만 HTML로 변환되며 어떤 서버로도 전송되지 않습니다. 사내 문서나 미공개 원고처럼 민감한 내용도 안심하고 미리 볼 수 있습니다. 다만 렌더링 라이브러리 코드 자체는 CDN에서 최초 한 번 불러옵니다.

자주 묻는 질문

Q. 표(table)나 체크리스트도 지원하나요?

A. 네, marked.js 라이브러리가 GitHub 마크다운과 유사한 확장 문법 대부분을 지원합니다.

Q. 이미지가 미리보기에 안 보이는 이유는 무엇인가요?

A. 마크다운의 이미지 문법은 접근 가능한 이미지 URL이 필요합니다. 로컬 파일 경로는 브라우저에서 직접 열 수 없습니다.

Q. HTML 복사 결과를 블로그 에디터에 붙여넣어도 되나요?

A. 대부분의 에디터는 HTML 붙여넣기를 지원하지만, 에디터별로 서식이 다르게 적용될 수 있습니다.
