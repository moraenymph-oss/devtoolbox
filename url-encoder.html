URL 인코더 / 디코더

한글·특수문자가 포함된 문자열을 URL에 안전하게 넣을 수 있도록 퍼센트 인코딩으로 변환합니다.

🔒 입력한 URL과 텍스트는 브라우저에서만 처리되며 서버로 전송되지 않습니다.

광고 영역

사용법

변환할 텍스트나 URL을 위 칸에 입력한 뒤 원하는 인코딩 방식을 선택하세요. 인코딩(Component)은 encodeURIComponent 방식으로 / ? = & 같은 URL 구분 문자까지 모두 변환하므로, 쿼리스트링의 값 하나를 인코딩할 때 사용합니다. 인코딩(전체 URL)은 encodeURI 방식으로 URL 구조는 유지한 채 한글과 공백 등만 변환하므로, 완성된 주소 전체를 인코딩할 때 적합합니다. 아래 칸에 인코딩된 문자열을 붙여넣고 디코딩을 누르면 원래 문자로 복원됩니다.

이런 상황에서 쓰세요

  • 한글 검색어가 포함된 URL을 API 요청이나 curl 명령에 넣어야 할 때
  • 로그에 찍힌 %EC%95%88%EB%85%95 같은 인코딩 문자열이 무슨 뜻인지 확인할 때
  • 카카오톡·슬랙 등에서 공유받은 긴 인코딩 URL의 실제 주소를 알고 싶을 때
  • 쿼리 파라미터에 &나 =가 포함된 값을 안전하게 전달해야 할 때
  • 리다이렉트 URL(returnUrl) 파라미터를 이중 인코딩해야 할 때

변환에는 브라우저 내장 함수인 encodeURIComponent, encodeURI, decodeURIComponent만 사용됩니다. 세션 토큰이 포함된 URL 등 민감한 주소도 외부로 전송되지 않으니 안심하고 변환하세요. 인코딩한 URL을 오프라인에서 공유하려면 QR코드 생성기도 참고하세요.

자주 묻는 질문

Q. encodeURIComponent와 encodeURI는 어떻게 다른가요?

A. encodeURIComponent는 /, ?, & 같은 URL 구분 문자까지 인코딩하고, encodeURI는 URL 구조는 유지한 채 한글 등 문자만 인코딩합니다.

Q. %20과 +는 같은 의미인가요?

A. 둘 다 공백을 나타내지만 %20은 URL 경로에, +는 주로 쿼리스트링(form 인코딩)에 쓰입니다.

Q. 이중 인코딩이 필요한 경우는 언제인가요?

A. 인코딩된 값이 다시 한 번 쿼리 파라미터로 전달될 때(예: 리다이렉트 URL의 returnUrl 파라미터)는 이중 인코딩이 필요할 수 있습니다.
