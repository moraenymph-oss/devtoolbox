UUID 생성기

암호학적으로 안전한 난수 기반의 UUID v4를 원하는 개수만큼 생성합니다.

🔒 브라우저의 crypto API로 생성되며 서버 통신이 없습니다.

광고 영역

사용법

생성 개수를 지정하고 생성 버튼을 누르면 UUID v4가 한 줄에 하나씩 만들어집니다. 대문자 옵션은 결과를 모두 대문자로, 하이픈 제거 옵션은 -를 뺀 32자리 문자열로 만들어 줍니다(일부 시스템은 하이픈 없는 형식을 사용합니다). 전체 복사 버튼으로 생성된 목록 전체를 클립보드에 복사할 수 있습니다. 이 도구는 브라우저의 crypto.randomUUID() / crypto.getRandomValues()를 사용하므로 예측 불가능한 안전한 난수로 생성됩니다.

이런 상황에서 쓰세요

  • 데이터베이스 기본 키나 외부 노출용 식별자가 필요할 때
  • 테스트 데이터·목업 데이터에 넣을 고유 ID를 여러 개 만들 때
  • API 요청 추적용 request-id, correlation-id 값을 임시로 만들 때
  • 파일명·리소스명 충돌을 피하기 위한 고유 문자열이 필요할 때
  • 라이선스 키, 초대 코드 등의 초안 값을 만들 때

UUID v4는 122비트의 무작위성을 가지므로 실무적으로 중복 걱정 없이 사용할 수 있습니다. 생성된 값은 화면에만 표시될 뿐 어디에도 저장·전송되지 않으므로, 새로고침하면 같은 값을 다시 볼 수 없다는 점만 유의하세요. 비슷한 방식의 안전한 난수 기반 값이 필요하다면 비밀번호 생성기도 사용해 보세요.

자주 묻는 질문

Q. UUID가 중복될 가능성이 있나요?

A. UUID v4는 122비트의 무작위성을 가져 실무적으로 중복될 확률이 거의 0에 가깝습니다.

Q. 하이픈을 제거해도 되나요?

A. 네, 일부 시스템은 하이픈 없는 32자리 형식을 요구하므로 옵션으로 제공합니다.

Q. 생성된 UUID가 저장되나요?

A. 아니요. 화면에만 표시되고 서버에 저장되지 않으므로, 새로고침하면 같은 값을 다시 볼 수 없습니다.
