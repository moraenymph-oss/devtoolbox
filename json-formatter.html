JSON 포매터 / 검증기

JSON을 붙여넣고 버튼 하나로 정렬(pretty print)하거나 압축하세요. 문법 오류도 바로 알려드립니다.

🔒 입력한 JSON은 브라우저에서만 처리되며 서버로 전송되지 않습니다.

광고 영역

사용법

위 입력창에 JSON 문자열을 붙여넣은 뒤 정렬(2칸/4칸) 버튼을 누르면 들여쓰기가 적용된 읽기 좋은 형태로 변환됩니다. 반대로 압축(Minify) 버튼은 공백과 줄바꿈을 모두 제거해 전송용으로 최소화된 한 줄 JSON을 만들어 줍니다. JSON에 문법 오류가 있으면 결과 대신 오류 메시지와 대략적인 위치가 표시되므로, 쉼표 누락이나 따옴표 오류를 빠르게 찾아 고칠 수 있습니다. 결과 복사 버튼으로 변환 결과를 클립보드에 바로 복사할 수 있습니다.

이런 상황에서 쓰세요

  • API 응답이 한 줄로 압축되어 와서 구조를 파악하기 어려울 때
  • 서버 로그에 찍힌 JSON에서 특정 필드 값을 눈으로 확인해야 할 때
  • 설정 파일(package.json, tsconfig.json 등)을 수정하다 문법 오류가 났는데 어디가 문제인지 모를 때
  • 문서나 위키에 JSON 예시를 깔끔하게 정리해서 넣고 싶을 때
  • 네트워크 전송량을 줄이기 위해 JSON을 최소화(minify)해야 할 때

사내 API 응답처럼 민감한 데이터가 포함된 JSON도 안심하고 사용할 수 있습니다. 이 도구는 자바스크립트의 내장 JSON.parse / JSON.stringify만 사용하여 여러분의 브라우저 안에서 모든 처리를 끝내며, 입력 내용은 어디에도 저장되거나 전송되지 않습니다. 실제 API 응답을 확인하는 절차가 궁금하다면 JSON 포매터로 API 응답 확인하는 법 가이드를 참고하세요.

자주 묻는 질문

Q. JSON 정렬과 압축의 차이는 무엇인가요?

A. 정렬은 들여쓰기를 넣어 가독성을 높이고, 압축(Minify)은 공백과 줄바꿈을 모두 제거해 전송 용량을 줄입니다.

Q. "Unexpected token" 같은 오류는 왜 발생하나요?

A. 쉼표 누락, 작은따옴표 사용, 마지막 항목 뒤 불필요한 쉼표(trailing comma) 등이 흔한 원인입니다. 오류 메시지의 줄·열 번호를 확인해 해당 위치를 고쳐보세요.

Q. 입력한 JSON이 서버에 저장되나요?

A. 아니요. 브라우저 안에서만 처리되며 입력 내용은 어디에도 저장되거나 전송되지 않습니다.
